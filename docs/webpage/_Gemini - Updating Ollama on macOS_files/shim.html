Ollama RAG Chatbot Project

🚀 Project Overview

This project brings together several cutting-edge technologies to create a functional chatbot. It can answer your questions by looking up information in your own local files, making it super smart and context-aware. Think of it as having a personalized AI expert for your documents!

✨ Key Features

  • Your AI, Your Machine: We use Ollama to run large language models (LLMs) directly on your computer. This means your data stays private, and you don't have to worry about external API costs.
  • Smart Answers with RAG: The chatbot doesn't just guess! It uses Retrieval Augmented Generation (RAG) to find relevant information from your documents before generating a response, making its answers much more accurate and informed.
  • Complete Application: This isn't just a piece of code; it's a full-fledged application with a user-friendly interface (built with React.js) and a powerful brain behind the scenes (a Python Flask server).
  • Organized and Easy to Understand: The project is designed with a clear separation between the frontend (what you see), the backend (the logic), and the AI services, making it easy to learn and modify.
  • Flexible AI Models: You can easily switch between different AI models (if you have them installed via Ollama) right from the chatbot's interface.
  • See the AI Think: There's a cool toggle that lets you see (or hide) the AI's internal "thought process" as it formulates its answers.

🧠 System Architecture: How It All Connects

Our chatbot system is like a well-oiled machine with different parts working together:


+-------------------+       +-----------------+       +-----------------+       +-----------------+
|   React Frontend  |       |  Flask Backend  |       |   Ollama Server |       |   ChromaDB      |
| (User Interface)  | <---> | (API & Logic)   | <---> | (LLM Execution) | <---> | (Vector Store)  |
+-------------------+       +-----------------+       +-----------------+       +-----------------+
        |                               |                     |                       |
        |                               |                     |                       |
        | (User Input/Display)          | (API Requests/Responses)  | (LLM Inference)       | (Vector Search)
        |                               |                     |                       |
        +-----------------------------------------------------------------------------------------+
                                        |
                                        | (Your Local Knowledge Base: ./data/*.txt)
                                        |
                                        +----------------------------------------------------------+

Components Explained:

  • React Frontend (`src/`):

    Conceptual Role: This is the face of our application, what you see and interact with. Think of it as the chatbot's dashboard and display screen. It's built to look good and be easy to use.

    It handles everything you see: the chat messages, the input box, and the model selection. It sends your questions to the Flask backend.

  • Flask Backend (`python/api.py`):

    Conceptual Role: This is the "brain" or "coordinator" of our application's logic. It's like the chatbot's personal assistant that takes your request, figures out what needs to be done (like talking to the AI or searching the knowledge base), and then sends the results back.

    It's a Python web server that acts as the middleman. It has special "endpoints" (like web addresses) where the React app sends its requests. It also uses LangChain to manage the RAG process.

  • Ollama Server:

    Conceptual Role: This is our local "AI engine." It's a separate program that runs on your computer and is responsible for the heavy lifting of understanding language and generating human-like text. It's where the actual LLM models live and run.

    The Flask backend sends requests to Ollama to get the AI's responses.

  • LangChain:

    Conceptual Role: Think of LangChain as the "master builder" that connects all the pieces of our AI system. It provides the tools and blueprints to make the AI smart enough to use our knowledge base effectively.

    It's a powerful framework used within the Flask backend. It helps with: Loading your documents, Breaking them down, Creating "memory" (Embeddings), Searching the "memory" (Vector Store), and Putting it all together for the LLM.

  • ChromaDB:

    Conceptual Role: This is our "smart library" or "memory bank" for the AI. Instead of just storing raw text, it stores those special, searchable numerical codes (embeddings) of your documents. This allows the AI to quickly find the most relevant information for your questions.

    It's a lightweight database specifically designed for these "vector" (numerical code) searches.

🧩 Conceptual Overview of React Components

The React frontend is built from several components that work together. Here's what each one does:

  • `App.js` (The Main Component):

    Conceptual Role: This is the "CEO" of the frontend. It manages the most important information (the "state") and makes all the key decisions.

    Responsibilities: It holds the entire chat history, tracks the selected AI model, manages loading states, contains the main logic for sending messages, and renders other components.

  • `ModelSelector.js` (The Sidebar):

    Conceptual Role: This is the "settings panel" of our chatbot. It's a specialized component that only cares about displaying the available AI models and letting the user choose one.

    Responsibilities: Displays models, allows selection, and contains the "Show Thinking" toggle. It communicates user choices back to `App.js`.

  • `ChatInterface.js` (The Chat Window):

    Conceptual Role: This is the "main display and input area." Its job is to show the conversation and provide a way for the user to type new messages.

    Responsibilities: Displays chat history, manages user input, and triggers the message sending logic in `App.js`.

  • `ollamaApi.js` (The API Utility):

    Conceptual Role: This is our "communications department." It's not a visual component, but a utility file that handles all the technical details of talking to our Flask backend.

    Responsibilities: Contains functions for making HTTP requests to the Flask API, abstracting network communication details from React components.

🔄 Detailed Use Case: How Your Question Becomes an Answer

Let's trace your question's journey from your keyboard to the chatbot's reply:

  1. You Ask a Question (React Frontend): You type your question and hit send. Your `App.js` immediately displays your message and a temporary "Thinking..." message.
  2. Your App Calls the Backend (React Frontend): Your React app sends just your latest question to the Flask backend via `ollamaApi.js`.
  3. The Backend Gets Your Question (Flask Backend): The Flask server (`api.py`) receives your question.
  4. The AI Gets Smart (Flask Backend with LangChain, Ollama, ChromaDB):
    • Finding Context: LangChain uses ChromaDB to find relevant information from your `data` files (Retrieval).
    • Generating the Answer: LangChain combines your question with the found context and sends it to the Ollama Server. Ollama's LLM generates the detailed answer.
  5. The Answer Comes Back (Flask Backend to React Frontend): The Flask backend sends the generated answer back to your React app.
  6. Your App Shows the Answer (React Frontend): Your React app receives the AI's answer. If "Show Thinking" is unchecked, it removes the `` block. Finally, it updates the "Thinking..." message with the full, polished AI response.

🛠️ Setup and Installation: Get Ready to Chat!

Follow these steps to get the project running on your local machine.

Prerequisites

  • Python 3.9+: [Download Python here](https://www.python.org/downloads/).
  • Node.js (LTS version) & npm: [Get Node.js here](https://nodejs.org/en/download/).
  • Ollama: [Download Ollama here](https://ollama.com/download).

Backend Setup (Python/Flask)

  1. Navigate to the `python` directory: 
    cd path/to/your/project/ollama-chatbot/python
  2. Create a Python Virtual Environment: 
    python3 -m venv venv
  3. Activate the Virtual Environment: 
    # macOS/Linux:
source venv/bin/activate

# Windows (Command Prompt):
venv\Scripts\activate.bat

# Windows (PowerShell):
.\venv\Scripts\Activate.ps1
  4. Install Python Libraries: 
    pip install Flask Flask-Cors langchain-community langchain-ollama chromadb requests
  5. Prepare Your Knowledge Base: 
    mkdir data

    Place your text files (e.g., `company_info.txt`) into this `data` folder.

Frontend Setup (React)

  1. Navigate to the main project directory: 
    cd path/to/your/project/ollama-chatbot
  2. Install Node.js Libraries: 
    npm install

Ollama Setup

  1. Start the Ollama Server: 
    ollama serve
  2. Download the AI Model: 
    ollama pull deepseek-r1

    Update `LLM_MODEL_NAME` in `api.py` if you use a different model.

▶️ How to Start the Project: Let's Get It Running!

Once everything is set up, starting your chatbot is a breeze:

  1. Start Ollama Server: 
    ollama serve

    Keep this terminal window open.

  2. Start the Flask Backend:
    • Open a new terminal window.
    • Navigate to the `python` directory: `cd path/to/your/project/ollama-chatbot/python`
    • Activate your virtual environment.
    • Run the Flask app: 
      python api.py

    Keep this terminal window open. The server will run on `http://127.0.0.1:5001`.

  3. Start the React Frontend:
    • Open a third new terminal window.
    • Navigate to the project root directory: `cd path/to/your/project/ollama-chatbot`
    • Start the React development server: 
      npm start

    Your browser should automatically open to `http://localhost:3000`.

📚 Key Libraries and Technologies: The Tools We Used

This project brings together a fantastic set of tools from both the Python and JavaScript worlds:

Python (Backend)

  • Flask: A super lightweight and flexible web framework for building the API.
  • Flask-Cors: Handles Cross-Origin Resource Sharing (CORS) to allow frontend-backend communication.
  • LangChain: The orchestrator for our AI logic, connecting LLMs, vector stores, and other components.
  • ChromaDB: A vector database used to store and retrieve document embeddings for RAG.
  • Requests: A popular HTTP library for making requests to the Ollama server.

JavaScript (Frontend)

  • React: The leading JavaScript library for building dynamic and interactive user interfaces.
  • react-bootstrap: Integrates Bootstrap components with React for pre-styled UI elements.
  • bootstrap: The foundational CSS framework for responsive and modern design.
  • `ollamaApi.js` (Custom Utility): A custom JavaScript file that encapsulates API calls to the Flask backend.

🚨 Troubleshooting: Common Issues and Solutions

If you run into any bumps along the road, here are some common issues and how to fix them:

  • "Port 5000/5001 already in use":

    Cause: Another program is using the port. Solution: Identify and kill the process (`sudo lsof -i :`, then `kill -9 `), or change the port in `api.py` and `ollamaApi.js`.

  • "Failed to connect to Ollama server":

    Cause: Ollama application not running or inaccessible. Solution: Ensure Ollama is running (`ollama serve`) and its port (11434) is not blocked.

  • "Model not found" or LLM errors in `api.py` console:

    Cause: The specified LLM model has not been pulled into Ollama. Solution: Run `ollama pull `.

  • React app is blank or not updating after sending a message:

    Cause: JavaScript errors or incorrect API response handling. Solution: Check browser console/network tab for errors. Ensure `generateResponse` parses JSON and `App.js` updates state correctly.

  • CORS errors:

    Cause: Browser blocking requests from React to Flask. Solution: Ensure `Flask-Cors` is installed and `CORS(app)` is initialized in `api.py`.
